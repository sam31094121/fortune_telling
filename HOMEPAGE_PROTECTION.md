# 🛡️ 首頁防護系統 - Homepage Guardian System

## 概述

這是一個**自動化的首頁監控和恢復系統**，確保你的首頁永遠不會消失。

### 核心承諾

✅ **首頁永遠可訪問** - 24/7 監控  
✅ **自動化恢復** - 遇到問題自動修復  
✅ **零人工介入** - 不需要問「怎麼又不見了」  
✅ **完整日誌** - 所有事件都記錄在案

---

## 🚀 如何使用

### 方法1：Windows 批處理（推薦）
雙擊 `start-protected.bat` 即可啟動完整防護

```bash
start-protected.bat
```

### 方法2：npm 命令
```bash
npm run dev:safe
```

### 方法3：僅運行衛士（已有 dev server 時）
```bash
npm run dev:guardian-only
```

## 📋 系統工作原理

### 監控流程

```
每10秒檢查一次
    ↓
訪問 http://localhost:3000
    ↓
檢查是否返回正常內容（天地人/人格）
    ├─ ✅ 正常 → 繼續監控
    └─ ❌ 失敗 → 記錄失敗
         ↓
    失敗3次以上？
    ├─ 否 → 繼續監控
    └─ 是 → 觸發自動恢復
         ↓
    自動恢復流程：
    1. 殺死所有現存 Node/npm 進程
    2. 驗證並修復配置（移除危險設置）
    3. 清理舊構建文件
    4. 重新啟動 dev server
    5. 等待服務啟動（最多30秒）
    6. 驗證首頁是否恢復
         ↓
    恢復成功？
    ├─ ✅ 是 → 返回監控，重置失敗計數
    └─ ❌ 否 → 記錄錯誤，保持監控
```

### 防護的場景

這個系統可以自動恢復以下問題：

| 問題 | 原因 | 恢復方式 |
|------|------|---------|
| 首頁 404 | 構建失敗 | 重新構建 |
| 無法連接 | dev server 崩潰 | 重新啟動 |
| 配置錯誤 | optimizeCss 等危險設置 | 自動修復 |
| 端口衝突 | 多個進程佔用 | 強制清理 |
| 緩存污染 | .next 文件夾損壞 | 清理重建 |

## 🔒 核心保護

### 什麼不會再發生

❌ 首頁突然 404  
❌ 無法連接 localhost:3000  
❌ 配置錯誤導致構建失敗  
❌ 需要手動重啟 dev server  
❌ 問題持續多次而不恢復  

### 系統會自動做的事

✅ 每10秒檢查首頁狀態  
✅ 失敗3次後自動恢復  
✅ 修復配置問題  
✅ 清理進程衝突  
✅ 重建損壞的文件  
✅ 完整日誌記錄所有事件  
✅ 確保首頁永遠可訪問

```javascript
const CHECK_INTERVAL = 10000;  // 檢查間隔（毫秒）
const MAX_RETRIES = 3;         // 失敗多少次後觸發恢復
```

## 📝 技術細節

### 使用的技術

- **Node.js child_process** - 進程管理
- **http 模塊** - 健康檢查
- **fs 模塊** - 文件操作和日誌
- **concurrently** - 並行運行多個進程

### 系統要求

- Node.js 14+ ✅
- npm ✅
- Windows/Mac/Linux ✅

## ⚠️ 重要提示

1. **不要停止衛士** - 除非你確實想停止保護
2. **日誌會累積** - 定期檢查 `.guardian-log.txt` 的大小
3. **進程管理** - 衛士會自動殺死衝突的進程
4. **配置備份** - 衛士修改配置時會自動恢復
