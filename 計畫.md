# 血型生日配對分析網頁 — 完整計畫

> **AI 核心：Google Gemini API**
> 標示 `🔴 CRITICAL` 的地方需要你做出選擇，才能繼續實作。

---

## 一、專案概覽

使用者在網頁輸入兩個人的血型與生日，系統自動計算星座，再把資料送給 Gemini AI 分析兩人的配對度，最後以視覺化分數卡呈現結果。

---

## 二、技術選型

| 層次 | 建議技術 | 說明 |
|------|----------|------|
| 前端框架 | Next.js 14 (App Router) | React 18、TypeScript、SSR/SSG |
| 樣式 | Tailwind CSS | 快速開發、響應式 |
| AI 後端 | Google Gemini API | `@google/generative-ai` SDK |
| 部署 | Vercel | 零設定、免費層 |

### 🔴 CRITICAL #1 — Gemini 模型選擇

目前 Gemini 有以下選項：

| 模型 | 速度 | 品質 | 費用 |
|------|------|------|------|
| `gemini-1.5-flash` | 最快 | 普通 | 最低（有免費額度）|
| `gemini-1.5-pro` | 中等 | 高 | 較高 |
| `gemini-2.0-flash` | 快 | 高 | 低 |

**你需要選擇：要用哪個模型？（建議先從 `gemini-2.0-flash` 開始）**

## 六、API 流程

```
使用者填表 → 前端計算星座
    ↓
POST /api/analyze
    ↓
Next.js Route Handler → 呼叫 Gemini API
    ↓
解析 JSON 回應 → 回傳給前端
    ↓
前端渲染分數卡
```

### 安全設計
- **API Key 放在 `.env.local`**，只在 server-side 使用
- 前端不直接呼叫 Gemini，只打自己的 `/api/analyze`
- 加入基本的 input 驗證（血型、日期格式）
